# AI 提示词

本节提供可直接喂给 AI 助手的 **系统提示词**，用于指导 AI 制作 Legado Tauri 书源。

## 设计理念

传统做法是将所有 API 文档一次性塞入提示词，导致：

- 上下文膨胀，AI 容易迷失重点
- 小说 / 漫画 / 视频三种类型的差异被淹没
- 高级 API（浏览器探测、图片处理）对大多数书源无用却始终占位

**渐进式披露**的核心思路：

```
阶段 1 → 环境约束 + 文件骨架          （所有书源都需要）
阶段 2 → 判断书源类型                  （小说 / 漫画 / 视频）
阶段 3 → 逐模块实现 + 按需查阅 API    （只看当前模块相关的文档）
阶段 4 → 高级 API 按需引入            （仅在遇到特殊站点时查阅）
阶段 5 → 收尾测试 + 最佳实践          （全部模块完成后）
```

AI 在每个阶段只需要查阅对应的文档页面，而非一次性加载全部内容。

## 使用方式

### 方式一：直接复制

将 [书源制作提示词](/prompt/booksource) 页面内容复制为 AI 对话的系统提示词。

### 方式二：作为项目指令

在 VS Code / Cursor 等编辑器中，将提示词文件放入项目的 `.github/copilot-instructions.md` 或 `.cursorrules`。

### 方式三：结合文档站

提示词中的文档链接指向本站页面。AI 如果支持联网，可以直接访问链接获取完整 API 细节。不支持联网时，提示词本身包含了足够的摘要信息。
